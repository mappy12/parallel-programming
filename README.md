# Лабораторная работа №3: Параллельное умножение матриц (MPI)

## Описание проекта
Данный проект реализует параллельное умножение квадратных матриц с использованием технологии **MPI (Message Passing Interface)** на языке C++. Проведена серия экспериментов по исследованию масштабируемости алгоритма на процессоре **Apple M1 (Silicon)** с использованием схемы распараллеливания *Master-Worker*.

В отличие от лабораторной работы №2 (OpenMP), где использовалась общая память, в данной работе каждый процесс имеет собственное адресное пространство, а обмен данными осуществляется посредством явной передачи сообщений.

## Техническая реализация

### Алгоритм
Используется классический алгоритм умножения матриц сложностью $O(N^3)$:
$$C_{ij} = \sum_{k=1}^{N} A_{ik} \times B_{kj}$$

**Схема распараллеливания (Master-Worker):**
1.  **Процесс 0 (Master):**
    *   Считывает матрицы $A$ и $B$ из файлов.
    *   Рассылает полную матрицу $B$ всем процессам через `MPI_Bcast`.
    *   Распределяет строки матрицы $A$ между процессами. Если количество строк $N$ не делится нацело на число процессов $P$, первые процессы получают на одну строку больше (учет остатка). Рассылка выполняется через цикл `MPI_Send`.
    *   Собирает вычисленные части результирующей матрицы $C$ через цикл `MPI_Recv`.
    *   Записывает итог в файл и выводит время.
      
2.  **Остальные процессы (Workers):**
    *   Принимают матрицу $B$.
    *   Принимают свою часть строк матрицы $A$.
    *   Выполняют локальное умножение.
    *   Отправляют результат процессу 0.

### Ключевые особенности реализации
*   **Непрерывность памяти:** Для корректной работы MPI данные хранятся в плоских векторах (`std::vector<double>`), что гарантирует передачу непрерывных блоков памяти.
*   **Гибкость:** Алгоритм корректно работает даже если размер матрицы не кратен количеству процессов.
*   **Замер времени:** Используется стандартный `std::chrono`, замер производится только над частью вычислений (после рассылки и до сбора данных), синхронизация через `MPI_Barrier`.

## Визуальный анализ

### 1. Зависимость времени от количества процессов

![](mpi_plot_time.png)

_Для малых матриц (N=200, 400) время снижается почти линейно. Для больших матриц снижение замедляется из-за роста времени на передачу данных._

### 2. Ускорение

![](mpi_plot_speedup.png)

_Для размеров N=200 и N=400 кривые практически совпадают с идеальной линией (ускорение ~7.3–7.4 при 8 процессах). Для N=2000 наблюдается отклонение вниз (ускорение 3.67), что указывает на влияние накладных расходов._

### 3. Эффективность использования ресурсов

![](mpi_plot_efficiency.png)

_Эффективность высока (>90%) для малых задач даже при 8 процессах. С ростом размера задачи эффективность падает до ~46%, так как время коммуникации начинает преобладать над временем вычислений._

## Анализ результатов и выводы

### 1. Высокая эффективность на малых данных

Удивительно, но именно для малых матриц (N=200, N=400) программа показала наилучшую масштабируемость:

- При **8** процессах ускорение составило **7.45×** и **7.34×** соответственно.
- Эффективность превысила **90%**.

**Причина:** Объем передаваемых данных невелик (например, матрица 400×400 занимает всего 1.2 МБ), поэтому накладные расходы на вызовы MPI_Send/MPI_Recv минимальны по сравнению с выгодой от распараллеливания вычислений.

### 2. Снижение эффективности на больших данных

Для матрицы **2000×2000** эффективность упала до **45.9%** при ускорении **3.67**×.

**Причина:** Объем данных возрастает до **~32 МБ**. В модели MPI эти данные необходимо физически копировать из памяти процесса-мастера в память рабочих процессов.

### 3. Сравнение с OpenMP

| Параметр | OpenMP (Потоки) | MPI (Процессы) | Разница |
| :--- | :---: | :---: | :--- |
| Макс. ускорение (N=2000, 8 ед.) | ~9.85× | ~3.67× | MPI медленнее из-за копирования данных. |
| Эффективность (N=2000, 8 ед.) | 123% | 45.9% | В MPI нет эффекта сверхлинейности. |
| Масштабируемость | Ограничена одной машиной | Масштабируется на кластеры | Главное преимущество MPI. |

**MPI** эффективен для распределенных систем, где память физически разделена. На одной машине для задач с плотными вычислениями и интенсивным обменом данными он проигрывает технологиям общей памяти (OpenMP) из-за накладных расходов на межпроцессную коммуникацию.

### 4. Итоговый вывод

1. Разработанная программа корректно реализует параллельное умножение матриц по схеме Master-Worker с использованием библиотек **MPI**.
2. Достигнуто ускорение до **7.45 раз** для малых матриц и **3.67 раз** для больших при использовании **8 процессов**.
3. Экспериментально доказано, что с ростом объема передаваемых данных эффективность MPI-программы на одноузловой системе снижается из-за затрат на коммуникацию.
4. Все тесты завершены успешно (статус **PASSED**), результаты верифицированы.

## Инструкция по запуску

### 1. Установка зависимостей

#### Для macOS

Требуется пакетный менеджер [Homebrew](https://brew.sh):

**Установка OpenMPI**

`brew install open-mpi`

#### Для Linux (Ubuntu/Debian)

`sudo apt update`
`sudo apt install libopenmpi-dev openmpi-bin -y`

### 2. Компиляция программы

`mpicxx -O2 -std=c++11 -o src/matrix_mpi src/main_mpi.cpp`

### 3. Запуск экспериментов

`python3 benchmarks_mpi.py`
